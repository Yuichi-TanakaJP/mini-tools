# 開示イベントレーダー UAT

## 前提

- `MARKET_INFO_API_BASE_URL` が開示イベントAPIを提供している。
- マイ銘柄に検証対象コードを保有またはウォッチとして登録している。

## 確認項目

- `/tools/disclosure-radar` で最新日のデータと優待変更タブが表示される。
- 初期表示が7日で、直近1日・7日・30日を切り替えられる。
- manifestレスポンスが`Cache-Control: public, max-age=300`を返す。
- 日付別レスポンスが`Cache-Control: public, max-age=31536000, immutable`を返す。
- 同じ日付別URLを再表示した場合、ブラウザキャッシュを利用できる。
- 新しい日付がmanifestへ追加された場合、既存日付を保持したまま新しい日付だけを取得できる。
- 直近1日は土日0件日ではなく、直近の非0件日を表示する。
- 優待変更タブはマイ銘柄の登録有無に関係なく全対象を表示する。
- マイ銘柄タブは登録コードに一致する `audience=personal` のイベントだけを表示する。
- 銘柄コード、会社名、タイトルの検索が機能する。
- 表示対象に応じた分類フィルターが機能する。
- 確認済みにしたカードが薄くなり、再読み込み後も状態が維持される。
- 未確認のみを有効にすると確認済みカードが除外される。
- `?event=<event_id>` で対象カードが強調表示される。
- 公開元リンクが新しいタブで開く。
- API未設定または取得失敗時にエラー案内が表示される。
- マイ銘柄が空の場合に登録画面へのリンクが表示される。
- 幅720px以下で下部ナビが表示され、本文と重ならない。
- 下部ナビのホーム、マイ銘柄、優待、カレンダー、開示が各画面へ遷移する。
