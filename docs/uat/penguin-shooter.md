# ペンギンシューター UAT チェックリスト

## 確認画面・URL

| 環境 | URL |
|---|---|
| 本番 | `https://mini-tools-rho.vercel.app/tools/penguin-shooter` |
| Preview | Vercel PR コメントの URL + `/tools/penguin-shooter` |
| ローカル | `http://localhost:3000/tools/penguin-shooter` |

## データ取得の仕組み（確認の前提知識）

- 初期版は外部 API を使わない
- ゲーム状態はプレイ中のメモリ上で管理する
- 初期版では永続保存なし
- 画像アセットを使う場合でも、画像読み込み失敗でページ全体がクラッシュしないことを確認する

## 正常系チェックポイント

### 初回表示

- [ ] ページが正常表示される
- [ ] タイトル「ペンギンシューター」が表示される
- [ ] 開始ボタンが表示される
- [ ] 既存の `/tools/penguin-rabbit-shooter` とは別 URL として開ける

### オープニング

- [ ] 開始後、10秒オープニングが表示される
- [ ] Shuty 登場、Pen 登場、Shoot 捕獲、コイン強化、出発の流れが分かる
- [ ] オープニングに 1/5 から 5/5 のカット進行が表示される
- [ ] オープニングをスキップできる
- [ ] スキップ後にゲームプレイへ進む

### PC 操作

- [ ] 矢印キーでプレイヤーを移動できる
- [ ] Space でショットを撃てる
- [ ] プレイヤーがゲームフィールド外へ出ない
- [ ] 敵にショットが当たると撃破できる
- [ ] 敵または敵弾に当たるとライフが減る、またはゲームオーバーになる

### スマホ操作

- [ ] タッチ操作パネルが表示される
- [ ] タッチ操作で移動できる
- [ ] タッチ操作でショットを撃てる
- [ ] 操作ボタンが重ならない
- [ ] 操作ボタンのラベルやアイコンがはみ出さない

### ゲーム進行

- [ ] スコアが増える
- [ ] コインを獲得できる
- [ ] 武器強化状態が分かる
- [ ] 3-Way Spread Shot が解放または使用できる
- [ ] ボムを使用できる
- [ ] ボム使用後に残数が減る、または再使用できない
- [ ] Stage 1-4 で撃破目標数を満たすと次ステージへ進む
- [ ] Stage 3 到達時、Maro-kun の補給メッセージが表示される
- [ ] Stage 5 で捕獲UFOを倒すと Shoot 救出演出へ進む
- [ ] 捕獲UFO、Shoot の捕獲表示、SUPPORT 表示が画面内で確認できる
- [ ] ステージ別ゲージと全体救出ゲージが破綻しない

### クリア / ゲームオーバー

- [ ] Shoot 救出時にクリア表示になる
- [ ] ゲームオーバー時にスコアとリトライ導線が表示される
- [ ] リトライで初期状態から再開できる

## レイアウトチェック

| Viewport | 確認内容 |
|---|---|
| Desktop 1280x720 | HUD、ゲームフィールド、操作説明が重ならない |
| Desktop 1024x768 | ゲームフィールドが狭くなっても主要UIが収まる |
| Mobile 390x844 | タッチ操作パネル込みで主要UIが操作できる |
| Mobile 430x932 | タッチ操作パネルのボタンが重ならない |

## 異常系チェックポイント

| シナリオ | 期待する挙動 |
|---|---|
| オープニング中にスキップを連打 | ゲーム開始状態が二重にならない |
| ショットキーを押しっぱなし | レート制限され、描画や操作が破綻しない |
| タッチ操作を押しっぱなし | 移動が継続し、指を離すと止まる |
| 画像アセット読み込み失敗 | 最低限の代替表示でページが継続する |
| ページリロード | タイトルまたは初期状態に戻る |

## 環境ごとの注意点

| 環境 | 注意点 |
|---|---|
| 本番 | スマホ実機でタッチ操作とレイアウトを確認する |
| Preview | PR ごとの変更範囲に応じて、オープニング・操作・クリア条件を重点確認する |
| ローカル | DevTools console に hydration mismatch や runtime error が出ていないか確認する |
