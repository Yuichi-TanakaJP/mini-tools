# 優待ダッシュボード UAT

対象: `/tools/yutai-dashboard`
前提: PC の Chrome で確認する（スマホ最適化は対象外）。premium 認証が設定済みであること。

## 正常系

1. 未ログインで `/tools/yutai-dashboard` を開くと、データを表示せず `/premium/login` へ移動する
1a. premium ログインすると `/tools/yutai-dashboard` に戻り、ダッシュボードを開ける
1a-1. `?month=all` または月指定付きURLからログインした場合、ログイン後も同じ対象月へ戻る
1b. ログイン後30日以内は再ログインなしで開ける。30日を超えたセッションは再ログインになる
2. 対象月の候補銘柄がテーブルに一覧表示される（コード / 銘柄 / 権利月 / 簡易効率 / 日興 / SBI / 仕込み開始 / 1株開始 / クロス戦略 / 実績 / 操作）
3. 対象月を切り替えると一覧が更新され、URL の `?month=` が変わる
4. 対象月「全月」を選ぶと全月の候補が結合表示される（同一銘柄は権利月ごとに1行）
5. ☆ でピックすると行が琥珀系になり、`/tools/yutai-candidates` の同銘柄にもピックが反映される
6. ✕ でパスすると行が灰色になり、ピックとは排他になる
7. 「＋メモ」で優待メモへ追加でき、行が緑系（追加済）になる。`/tools/yutai-memo` に銘柄が現れる
8. 行クリックで右に詳細パネルが開き、優待内容・リンク・日興規制明細・優待メモ・クロス購入実績が表示される
9. 優待メモ登録済み銘柄では、仕込み開始（◯ヶ月前）・1株開始・クロス戦略・実績（✓件数）が表に出る
10. 表示軸を「仕込み月で探す」に切り替えると、仕込み時期を設定した登録銘柄だけが表示される（全月選択時は仕込み時期設定済みの全銘柄）
11. 状態 / 日興クロス / SBI / クロス戦略 / 検索 / 並び順のフィルタが機能する
12. 登録済み銘柄の詳細パネルで「編集」を押すとフォームが開き、クロス戦略・仕込み開始・1株保有開始・メモ本文などを保存できる。表の該当セルと `/tools/yutai-memo` に反映される
13. 編集中に別の行を選ぶ・詳細を閉じる・対象月を変えると編集はキャンセルされる
13a. 詳細パネルの「メモから外す」で、誤って追加した優待メモを解除できる（確認ダイアログあり）。複数権利月のメモは対象月だけ外れ、単月メモは削除される。表とメモ帳に反映される
14. 「クロス戦略 / 仕込み開始 / 1株開始」セル（破線枠）をクリックするとその場で編集でき、選択・入力確定で保存される。表の値と `/tools/yutai-memo` に反映される
15. 未登録の候補行でもこれらのセルを編集でき、確定すると自動で優待メモへ追加され、通知が出て行が緑（追加済）になる
16. 未設定のセルは 仕込み開始「未設定」・1株開始「未購入」・クロス戦略「未設定」と表示される（`-` ではない）
17. 1株開始セルをクリックすると、その場に浮かぶ月ピッカー（年送り＋12ヶ月グリッド）が開き、月を1タップで選べる。セルのレイアウトは変わらない。「未購入に戻す」で未設定へ戻せる。ピッカー外クリック・Escape・スクロールで閉じる
18. 「12ヶ月ビュー」タブに切り替えると、登録済み銘柄が 銘柄 × 12 ヶ月グリッドで表示される。仕込み開始（仕）〜権利月（権）が帯で描かれ、取得済み権利月に ✓（緑）が付く。凡例が表示される
19. 12ヶ月ビューで検索・クロス戦略フィルタが効く。登録済み銘柄が無いときは空状態メッセージが出る。横スクロールしても銘柄名列は固定される
20. 12ヶ月ビューに年度セレクタがあり、既定は今年。年度を変えると取得✓と1株保有の表示が切り替わる。権利月・仕込みは年度を変えても同じ
21. 取得済みは、選択年度に取得した権利月が緑✓、選択年度以外に取得実績がある月が灰✓（ツールチップに取得年）。今年に取ったか／過去に取ったかが区別できる
22. 1株保有は選択年度で解釈される。開始年より後の年度は通年、開始年度は開始月〜12月（開始前は塗らない）、開始年度より前は非保有。銘柄名の下に「1株 YYYY-MM〜（N年目）」が出る
23. 選択年度が今年のとき、現在月（JST）の列が淡くハイライトされる。過去/未来年度では出ない
23a. 優待メモを取得済み（＝仕込み済み）にすると、12ヶ月ビューで実際に仕込んだ月に「仕込み実施」のフクシア●、対象の権利月に緑✓が出る。アーカイブ後も選択した権利年度で同じ表示を確認でき、月セルの手動クリック記録はできない。凡例に「仕込み実施」がある
24. 権利月は取得の有無に関わらず「権」で表示され、取得済みは右上✓、仕込み開始と同月なら左上に橙ドットが重なる
25. 同一銘柄が複数の権利月を持つ場合、テーブルは権利月ごとに別行になり、各行はその行自身の権利月を表示する（別月の行が同じ月に見えない）
26. 権利月軸の候補行を開き、必要株数と優待価値（円）を入力すると、株価取得済み銘柄は実株価で簡易優待効率・必要資金が計算され、詳細に「株価」と基準日、一覧に同じ率が表示される
26a. ヘッダーに株価の取得件数と生成時刻が表示される。対象銘柄の実株価がない場合は「概算株価」として最低投資金額÷100を使う
26b. 株価APIの正常レスポンスは`private, max-age=86400, stale-if-error=604800`かつ`Vary: Cookie`。要求月と`scope_month`の不一致、未ログイン、APIエラーは`private, no-store`になる
27. 再読み込み後も コード:権利月 ごとの必要株数と優待価値が復元され、別の権利月には混ざらない。入力を空欄に戻すと未設定にできる
28. 「簡易優待効率が高い順」を選ぶと、実株価または概算株価で計算可能な行が率の降順になり、必要な入力と両方の株価根拠がない行は末尾になる
29. 仕込み月軸の詳細では簡易効率を誤った権利月へ保存せず、「月別の候補行（権利月軸）から入力」と案内される

## 異常系

1. 未ログインまたは改ざんされたpremium Cookieではダッシュボードを表示しない
1b. 一度ログインして表示した後でも、ログアウト・期限切れ・オフライン時にService Workerのキャッシュからダッシュボードを再表示しない
1a. `MARKET_INFO_API_BASE_URL` 未設定（またはAPI障害）時、「データ未接続」表示になりページはクラッシュしない
2. LocalStorage が空（初回利用）でも一覧は表示され、メモ系列は `-` になる
3. 仕込み月軸で対象銘柄がない場合、「この月に仕込みを開始する登録銘柄はありません」と表示される
4. 候補データに存在しないコードの日興 / SBI 列は `-` になる
5. 日興 / SBI データが未取得のとき、対応するフィルタは「（データなし）」表示で無効化される
6. 必要株数・優待価値が未設定・0以下・不正値、または実株価と最低投資金額の両方がない場合、簡易効率はハイフンまたは計算待ち表示になりページはクラッシュしない
7. 株価APIが失敗または不正schemaでも候補一覧は表示され、最低投資金額がある銘柄は概算計算へフォールバックする。上流tokenやエラー本文は表示しない
8. 月が替わると株価APIの`month=YYYY-MM`が新しいJST年月になり、前月のHTTPキャッシュキーを再利用しない。ログアウト後もCookieが異なるため認証済みキャッシュを利用しない

## 環境ごとの注意

- production では repo 同梱 JSON への自動 fallback はない（非 production のみ）
- ピック / パス / メモは端末・ブラウザごとに独立する（LocalStorage）
