# Backlog インデックス

未着手の改善候補、PR 単位の候補、将来の検討事項を置く場所です。
現在仕様や確定判断は、必要に応じて specs または decision-log に移します。

| ドキュメント | 内容 |
|---|---|
| [yutai-expiry UI / data model backlog](./yutai-expiry-ui-data-model.md) | 優待期限帳の券種管理、日付入力、データモデル改善候補 |

## 更新ルール

- backlog は未確定の候補として扱う
- 実装方針が決まったら decision-log に判断理由を残す
- 現在仕様になった内容は tool spec へ反映する
