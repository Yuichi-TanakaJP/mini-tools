# 2026-04-25 premium 保有銘柄ダッシュボードの初期方針

## 背景

- mini-tools に、自分の保有銘柄や優待・配当・決算予定をまとめて確認する個人向けダッシュボードを追加する案が出た。
- 保有銘柄や取得単価は見られたくない個人データなので、通常の公開 tool ではなく premium 配下に置く方針を確認した。
- 初期実装では実個人データを扱わず、UI と集計の確認用にサンプルデータだけを同梱する。

## 今回決めたこと

- URL は `/premium/portfolio` とし、既存の premium cookie 認証を通したページにする。
- リポジトリに入れるのは開発確認用のサンプル保有データのみとする。
- 実個人データは当面リポジトリに入れない。将来は localStorage / IndexedDB / 暗号化ローカル保存 / サーバー保存のどれかを別途判断する。
- 初期画面は、評価額、含み損益、年間配当見込み、次のイベント、保有銘柄一覧、直近イベント、配分を表示する。

## 判断理由

- premium 配下にすることで、一般公開 tool と個人向け機能の導線を分離できる。
- ただし premium 配下に置くだけではデータ保護にならないため、実個人データを git 管理しない方針を先に固定する。
- サンプルデータで UI と集計の形を固めると、後から入力・保存・インポート機能を足しやすい。

## 影響範囲

- `app/premium/portfolio/`
- `app/premium/page.tsx`
- 将来の保存方式、インポート方式、premium 認証方式

## 残課題

- 実個人データの保存先を決める。
- 保有データの入力 UI、CSV インポート、バックアップ方式を決める。
- 既存の優待・決算・ランキング系データと突き合わせる ViewModel を設計する。
- premium 認証を本番向けの方式にするか判断する。
