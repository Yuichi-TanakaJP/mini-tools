# 2026-06-14 全ツール切り替え用ハンバーガー＋ドロワー

## 背景

- ツール数が 19 まで増え、ツール間を移動するには一度ホームへ戻る必要があった。
- 直前に追加したモバイル下部ナビ（[2026-06-13](./2026-06-13-disclosure-radar-and-mobile-nav.md)）は主要 5 項目のショートカットで、全ツールへは到達できない。
- ヘッダーのロゴ→ホームだけでは、どこからでも任意のツールへ 1 アクションで移動できない。

## 今回決めたこと

- ヘッダー左にハンバーガー（≡）を追加し、左からスライドインするドロワーで全ツールへ移動できるようにする。
- ドロワー内容: ツール検索ボックス、🏠ホーム、カテゴリ別ツール一覧（入力ツール / 優待・保有 / 市場データ / おまけ）。
- `usePathname()` で現在のツールを `aria-current="page"` でハイライトする。
- 背景クリック / × / Esc / リンク選択で閉じる。開いている間は body スクロールをロックする。
- 開いたらパネル（`tabIndex=-1`）へフォーカスを移し、閉じたら呼び出し元（ハンバーガー）へ戻す。検索 input ではなくパネルへ当て、モバイルでソフトキーボードが即立ち上がるのを避ける。
- アナリティクスはホームを `nav_home_clicked`、ツールを既存の `tool_opened`（`source: "nav_drawer"` 付き）に揃え、ToolGridClient とイベント名を統一する。
- ヘッダーのロゴ→ホームは Web 慣習として**維持**する（下部ナビ・ドロワーと役割が重複しても残す）。
- ツール定義は `lib/tools-catalog.ts` に一元化し、ホームのグリッドとドロワーが同じ定義を参照する。

## 判断理由

- 解くべき課題は「ホームへ帰れない」ことより「ツール間を直接移動できない」ことだったため、ホーム導線の置き換えではなくクイックスイッチャーの追加とした。
- 形式はドロップダウン・固定タブバーも検討したが、19 個（今後も増加）をカテゴリ＋検索で破綻なく収められる左ドロワーを採用した。
- 下部ナビ（モバイル主要 5 項目）とドロワー（全ツール・全デバイス）は役割が異なるため共存させる。
- ロゴ→ホームは利用者が無意識に期待する操作で、冗長でも残す方が迷いにくい。
- ドロワーはヘッダー（`position: sticky; z-index:10`）の stacking context に閉じ込めないよう、`<Header>` の兄弟として描画し、下部ナビ（z-index:30）より上（overlay 100 / panel 101）に出す。

## 影響範囲

- 追加: `components/NavDrawer.tsx` / `components/NavDrawer.module.css` / `lib/tools-catalog.ts`
- 変更: `components/Header.tsx`（ハンバーガー＋開閉 state、ドロワー描画）、`app/page.tsx`（ツール配列を catalog から import）
- テスト: `tests/ui-smoke/nav-drawer.spec.ts`（開閉・検索・遷移・ハイライト・Esc）
- 既存のホームグリッド表示・URL は不変。

## 残課題

- カテゴリ分けの粒度（市場データが多め）の見直し。
- 下部ナビ項目とドロワー主要項目の整合（将来ツールが増えたときの優先度）。
- 完全なフォーカストラップ（ドロワー内 Tab 循環）と背景の `inert` 化。現状はパネルへフォーカス移動＋閉時の復帰までで、Tab で背景へ抜けうる。
- `components/MobileBottomNav.tsx` の `ITEMS`（href/label）は独自定義のまま。href が catalog とドリフトしうるため、将来 catalog 由来へ寄せるか検討する。
