# 2026-07-12 12ヶ月ビューの仕込み実績記録

## 背景

12ヶ月ビューは「計画上の仕込み（`preparationMonthsBefore` から計算した帯・仕込み開始月）」と「取得済み（年度別の実績）」を表示していたが、**実際にいつ仕込んだか**の実績を記録・表示できなかった。取得済みだけが実績で、仕込みは計画のみだった。

## 決めたこと

- 12ヶ月ビューに **仕込み実績**（実際にクロス/現物を仕込んだ年月）を年度別に記録・表示する。
- データはダッシュボード独立の LocalStorage（キー `yutai_dashboard_prep_log_v1`）で per-銘柄に持つ。既存の優待メモ・期限帳のアーカイブには手を入れない（ピック/パスと同じ独立ステータス方式）。
- 記録単位は `code → 実施年月("YYYY-MM")[]`。12ヶ月ビューの月セルをクリックすると、選択年度のその月の仕込み実施をトグルする。
- 表示は計画（帯/仕）とは別マーカー（フクシアの●）で重ねる。凡例に「仕込み実施」を追加。
- 断念記録（取引停止・残数なしで見送った時期）は今回のスコープ外。別途仕様化する。

## 理由

- 一般信用クロスなどでは、いつ仕込んだか（在庫確保の実績）が運用上重要で、計画月とずれることも多い。計画（帯）と実績（●）を重ねると、年度ごとの実態が追える。
- 影響を局所化するため、共有アーカイブを変えずダッシュボード専用ステータスにした。断念記録まで一体化すると設計が大きくなるため、まず仕込み実績だけ先行した。

## データと互換性

- `PrepLog = Record<string, string[]>`（code → 昇順ユニークな "YYYY-MM"）。
- 不正な年月・空コードは読み込み時に除外。空配列の code は保持しない。
- 既存データへの影響なし（新規キー）。

## 影響範囲

- `docs/specs/tools/yutai-dashboard.md`、`docs/uat/yutai-dashboard.md`
