# 2026-07-12 優待ダッシュボード 12ヶ月ビューの年度軸

## 背景

12ヶ月ビュー（Phase 3）は「銘柄 × 12 ヶ月」で権利月・仕込み・取得実績・1株保有を表示していたが、性質の違う3つの時間軸を月のみで混在させていた。

- 権利月 / 仕込み開始 / 仕込み期間 = 毎年サイクル（年度非依存）
- 取得済み = 年度ごとのデータ（いつ取ったか）
- 1株保有 = 複数年にまたがり継続

実装は取得実績の `entitlementMonthKey`（YYYY-MM）から月だけを抽出して年を捨てており、「今年の権利をもう取ったか」が分からなかった。1株保有も「開始月〜12月」に固定描画され、開始年のパターンに固定されて翌年以降の通年保有を表現できなかった。

## 決めたこと

- 12ヶ月ビューに **年度セレクタ** を追加する（既定=今年 JST）。年度は毎年サイクルの権利月・仕込みと、年度別の取得・1株保有を1画面で整合させる基準にする。
- **取得済み**は選択年度の取得のみ緑✓で示す。選択年度以外に取得実績がある月は淡い✓（過去取得マーク）で履歴を残す。ツールチップに取得年を列挙する。
- **1株保有**は選択年度で算出する。
  - 選択年度 > 開始年 → 通年（1〜12月）保有
  - 選択年度 = 開始年 → 開始月〜12月保有（開始月に開始マーカー）
  - 選択年度 < 開始年 → 非保有（塗らない）
  - 開始が YYYY-MM で判定できないフリーテキスト → 年不明として通年保有扱い（開始マーカーなし）
  - 銘柄名脇に「1株 YYYY-MM〜」と、選択年度での経過年数「N年目」を表示する
- **権利月・仕込み**は従来どおり年度非依存（毎年同じ）で表示する。
- **現在月マーカー**を追加する。選択年度が今年のときだけ、現在月（JST）の列を淡くハイライトする（過去/未来年度では出さない）。
- **凡例を整理**する。クロスサイクル系（権 / 仕 / 帯 / 仕込みと権利が同月）・実績系（取得済み / 過去に取得）・長期保有系（1株保有 / 開始）でグループ化して並べる。

## 理由

- 実運用で最も知りたいのは「今年度、この銘柄の権利をもう取ったか／これからか」であり、年度を軸にしないと答えられない。データ（archive の YYYY-MM）は年を持っているので、捨てずに使う方が正しい。
- 権利月は毎年ほぼ同月で固定なので年度非依存のままでよい（ユーザー確認済み）。一方、取得と1株保有の起点は年度に依存するため、同じ12ヶ月グリッドに載せるには年度基準が要る。
- 現在月マーカーと N年目表示は、次の仕込み時期の把握と長期優待の年数管理という運用目的に直接効く。
