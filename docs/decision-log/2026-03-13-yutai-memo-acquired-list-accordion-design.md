# yutai-memo #42 取得リスト年月アコーディオン設計メモ

## 対象 Issue

- #42 UI(yutai-memo): 取得リストを年月アコーディオンで折りたたみ表示にする
- 関連: #43（本設計の年月判定ロジックを利用）

## 目的

- 取得リストを `YYYY-MM` 単位で折りたたみ表示し、日常利用時の視認性を上げる。
- #43 の月替わり提案機能で再利用できる年月ロジックを先に確定する。

## 設計決定

### 1. 年月の基準

- グルーピング基準は **取得年月（acquiredAt 由来）** とする。
- 表示キーは `YYYY-MM`（例: `2026-03`）で統一する。
- 日付文字列が不正なデータは後述の「不明」グループに入れる。

### 2. 開閉状態の保持

- 初期表示は **全グループ折りたたみ**。
- 開閉状態は **セッション内メモリのみ**（永続化しない）。
- 理由: 初回実装の複雑さを上げずに要件を満たせるため。

### 3. 並び順

- 年月グループは **新しい年月が上**（降順）。
- 各グループ内の明細も **新しい取得日が上**（降順）。

### 4. 空/不正データの扱い

- `acquiredAt` が欠損またはパース不可のデータは `不明` グループに集約。
- `不明` グループは常に最下部に表示。

## UI 仕様（最小）

- ヘッダー表示: `YYYY-MM (N件)` / `不明 (N件)`
- ヘッダークリックで開閉
- 開閉状態は矢印などで視覚化
- 初期はすべて閉じた状態

## 実装メモ

- 既存の取得リスト生成処理の直前で `Map<string, Item[]>` へグルーピングする。
- ソートはキー生成後に一度だけ行う（レンダー中の再計算を避ける）。
- #43 実装時は、月判定ロジックを共通 util 化して再利用する。

## 今回のスコープ外

- 「直近1か月のみ初期展開」などの拡張
- 開閉状態の localStorage 永続化
- 表示ラベルのローカライズ
