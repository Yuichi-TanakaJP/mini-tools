# 2026-05-10 ペンギンシューター ボス攻撃のステージ別設計

## 背景

- ボスから弾を発射する仕組みを追加した（武器位置から狙い撃ち弾を出す）
- 現状は中ボス・ステージボスとも全ステージ共通の攻撃ロジック（中ボス: 100フレーム間隔/2発、ステージボス: 78フレーム間隔/3発）になっている
- 大ステージごとに難度カーブを作るため、ボス攻撃の挙動をステージごとに変えたい
- 現状の `BossDefinition` には HP・速度・横移動などはあるが、攻撃パターンの定義がない

## 今回決めたこと

ボス攻撃を以下の4軸でステージごとに設計し、`BossDefinition` に攻撃パラメータを追加する。

### 設計4軸

1. **パターン** — 弾の出方の種類
2. **発射間隔（フレーム）** — 大きいほど低頻度
3. **弾速** — 1フレームあたりの移動距離
4. **同時発射数 / バースト構成** — 1発射イベントで何発出すか

### パターン定義

| パターンID | 説明 |
|---|---|
| `aimed-1` | 単発・プレイヤー狙い撃ち |
| `aimed-2` | 左右武器から2発・どちらもプレイヤー狙い |
| `aimed-3` | 左右武器+中央キャノンから3発・全弾プレイヤー狙い |
| `spread-2` | 左右武器から左右ナナメ拡散（プレイヤー追尾しない） |
| `aimed-spread-3` | 中央狙い撃ち + 左右±18°拡散の3way |
| `ring-4` | 上半円に4方向均等弾（プレイヤー位置非依存） |
| `aimed-ring-6` | 中央狙い撃ち1 + 5方向リング |
| `burst-3` | 単武器から短間隔で3連射（バースト中は12フレーム間隔） |
| `cannon-burst-5` | 中央キャノンから5連射 + 左右ウェポンから狙い撃ち1発ずつ |
| `aimed-2-alt` | 左右ウィングから交互に2発（ワープ前提） |
| `aimed-ring-9` | 中央狙い撃ち1 + 8方向リング、HP50%以下で弾速+10% |

### データ仕様

`stageData.ts` の `BossDefinition` に以下を追加する。

```ts
type BossAttackPattern =
  | "aimed-1"
  | "aimed-2"
  | "aimed-3"
  | "spread-2"
  | "aimed-spread-3"
  | "ring-4"
  | "aimed-ring-6"
  | "burst-3"
  | "cannon-burst-5"
  | "aimed-2-alt"
  | "aimed-ring-9";

type BossAttack = {
  pattern: BossAttackPattern;
  fireInterval: number; // フレーム
  bulletSpeed: number;
  burstSize?: number;       // バースト系の発射数
  burstInterval?: number;   // バースト中の間隔（フレーム）
};

// BossDefinition に attack: BossAttack を追加
```

## 判断理由

- 4軸（パターン/間隔/弾速/同時発射数）に絞ることで、ボス10体の差を表現しつつ、実装ロジックを11パターン（enum）の switch で済ませられる
- 「狙い撃ちのみ」「拡散のみ」「リング状」「バースト連射」という4つの基礎挙動を組み合わせる構成にしたので、新ステージを足すときは既存パターンの再利用かパターン追加かが判断しやすい
- HP・武器ラベルなどの既存パラメータと別に `attack` をぶら下げることで、ビジュアル定義（`visual`）と攻撃ロジックを独立に変更できる
- ステージ1〜2は単発・低速で操作練習、ステージ3でリング初出、ステージ4でバースト初出、ステージ5でリングとバーストの複合という、機能を1つずつ足していく難度カーブにした
- ステージボスの中央キャノンは現状の3D表示でも目立つので、`cannon-burst-5` `aimed-ring-9` のようにキャノン由来のパターンを終盤に置いた

### 採らなかった案

- **HPトリガーの完全別パターン化**（HP50%で別の弾幕に切り替え） — 状態管理が増えるため、現段階は `aimed-ring-9` の弾速ブーストのみで段階的難化を表現する
- **ボスごとに完全自由な弾幕スクリプト** — 表現幅は広がるが、データだけで完結しなくなる。`BossAttackPattern` enum の組み合わせで足りる範囲に留める
- **常に間隔ランダム化** — 練習しづらいので、フレーム固定で再現性を保つ

## 影響範囲

- `app/tools/penguin-shooter/stageData.ts` — `BossDefinition` に `attack` フィールド追加、各ボスに値を入れる
- `app/tools/penguin-shooter/ToolClient.tsx` — 現状のハードコードされた狙い撃ち発射ロジックを `attack.pattern` 分岐に置き換え、バースト用の補助カウンタを `Enemy` に追加
- `docs/specs/tools/penguin-shooter.md` — ボスデータ仕様にステージ別攻撃設定を追記
- `docs/uat/penguin-shooter.md` — 各ボス戦の確認観点（攻撃パターン）の追記候補
- 互換性: 既存の HP・スコア・速度・横移動・ビジュアルは変更しない。攻撃挙動のみ変わる

## 残課題

- 実装フェーズで実フレームレート（60fps前提）でのプレイ感を確認し、バースト系の `burstInterval` を要調整
- `aimed-ring-9` のHP50%以下ブーストは、本当に必要かをプレイテストで判断する
- 弾の見た目は現状の warm/cool/white の3配色のまま流用するが、リング弾のみ専用配色にする案は要検討
- BGM・効果音をパターンごとに分けるかは別判断
