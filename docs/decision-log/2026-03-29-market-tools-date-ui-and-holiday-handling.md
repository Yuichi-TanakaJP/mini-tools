# 2026-03-29 market tools の日付 UI と休場日扱いの整理

## 背景

`earnings-calendar`、`stock-ranking`、`nikkei-contribution` で日付まわりの UI と休場日の扱いが揺れていた。  
今回の調整では、見た目の統一だけでなく、「どの tool が休場日を見せるか / 見せないか」も明文化しておく必要があった。

## 現時点の仕様

### 1. 決算カレンダー

- 月ナビゲーションを主導線にする
- 初期フォーカス日は、表示対象月に今日が含まれる場合は JST の今日を優先する
- `manifest.as_of_date` は更新日表示や fallback に使う
- JPX 休場日は **表示する**
- 休場日はカレンダーセルとして見せ、空状態には `JPX休場日です` を表示する
- 日別一覧の見出し部分には、追加の日付セレクトを置かない

### 2. 株価ランキング

- 日付ナビは「前日 / セレクト / 翌日」の 3 点構成にする
- 見た目は決算カレンダー系の白カード、丸い矢印ボタン、中央寄せセレクトに寄せる
- `manifest.dates` はそのまま UI に出さず、サーバー側で表示候補を絞る
- JPX 休場日と土日は **表示候補から除外する**
- 初期表示日は、フィルタ後の先頭日付を使う

### 3. 日経225寄与度

- 日付ナビは「前日 / セレクト / 翌日」の 3 点構成にする
- 見た目は株価ランキングとそろえる
- JPX 休場日と土日は **表示候補から除外する**
- 先頭側に混ざったゼロ変化日も除外対象にする

## tool ごとの差分整理

- `earnings-calendar` は「休場日もカレンダー情報として見せる」
- `stock-ranking` と `nikkei-contribution` は「取引日だけを選択候補として見せる」

この差分は意図的な仕様差であり、現時点では統一しない。

## タップハイライトの判断

- モバイルブラウザのタップハイライトは、個別コンポーネントで抑制しない
- [`app/globals.css`](../../app/globals.css) に `html { -webkit-tap-highlight-color: transparent; }` を置いて一括で抑制する
- 今後も同種の挙動は、まず global CSS で扱う
