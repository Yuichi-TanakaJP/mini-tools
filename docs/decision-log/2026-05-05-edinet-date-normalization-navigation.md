# EDINET: 無効日指定時の日付正規化とナビゲーション継続

## 背景

EDINET書類一覧で `?date=` に manifest 未掲載日を指定した場合、表示日の index が見つからず、矢印ナビゲーションがグレーアウトして移動しづらくなる事象があった。

## 決めたこと

- 表示日は `manifest.dates` を基準に正規化する。
  - 指定日が有効ならそのまま使用。
  - 無効なら「指定日以前で最も近い日」を優先し、見つからない場合は最新日を使用。
- 無効日から正規化が発生した場合は、URLを正規化先の日付へ `replace` で同期する。
- UIに「指定日のデータがないため直近開示日を表示中」という注記を表示する。

## 理由

- 日付ナビの基準を常に manifest 上の有効日へ揃えることで、矢印移動が停止する状態を回避できる。
- URLと実際の表示日を同期し、再読み込みや共有時の再現性を上げる。
- ユーザーが空日を指定したことによる挙動変更を理解しやすくする。

## 影響範囲

- `app/tools/edinet-documents/ToolClient.tsx` の日付算出・遷移ロジック。
- EDINET書類一覧の空日指定時の表示と導線。

## 追記（2026-05-05）

- `manifest.dates` の並び順はAPI都合で変わり得るため、クライアント側で日付文字列を降順ソートしてから正規化・ナビ判定に使う。
- これにより、配列が昇順で返却された場合に初回表示が最古日へ寄ってしまう不具合を防ぐ。
