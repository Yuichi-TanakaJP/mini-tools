# 2026-05-11 ペンギンシューター 古代遺跡とバリアアイテム

## 背景

- 終盤ボスのレーザーや誘導弾追加で被弾しやすくなったため、防御手段を増やしたい
- 「稀に遺跡の建物が途中で出てきて、その中からバリアのアイテムが出る。バリアは 3 秒、クールタイム 5 秒、一度発動すると継続」という要望が出た

## 今回決めたこと

### 古代遺跡（ruins 障害物）

- 既存の破壊可能 / 破壊不可能障害物に、3 種類目「古代遺跡」を追加する
- 出現率は obstacle spawn の roll < 0.07（約 7%）かつ画面上に既に遺跡が無いときのみ
- HP 3（通常破壊可能は HP 2）、落下速度は通常より遅め
- ビジュアルは紫発光、アイコン `🏛️`
- 破壊スコアは 220（通常破壊は 80）
- 破壊時にバリアアイテムを 1 個ドロップする

### バリアアイテム

- 落下速度 2.6（通常コインと同程度）
- Shuty に接触で取得
- 取得時のバリア状態は 3 種類:
  - idle: 発動して `BARRIER_ACTIVE_FRAMES` = 180 (3 秒) のカウントダウン開始
  - active: 何も起きない（アイテムは消費される）
  - cooldown: 何も起きない（アイテムは消費される）

### バリア発動中

- `BARRIER_ACTIVE_FRAMES` 残っているあいだ、`playerHit` を受けてもライフを減らさず `GUARD` の burst を出す
- 対象: 敵接触・障害物接触・敵弾・レーザー・誘導ミサイル すべて
- Shuty 周りに青いリング表示

### クールタイム

- active が 0 になった直後に `BARRIER_COOLDOWN_FRAMES` = 300 (5 秒) を開始
- クールタイム中はアイテム取得しても発動しない
- HUD に `🛡️ 残秒` または `🛡️ CD 残秒` を表示

## 判断理由

- ボムは既存だが「もう、どうにでもなれボム」で全敵掃討する別役割。バリアは「攻撃を続けながら一時無敵」というコンセプトの差別化
- ストック不可（クールタイム中の取得は捨て）にしたのは、ストックを許すと終盤に貯めて連発できてバランスが崩れるため
- 出現率 7% は obstacle spawn 1 個あたりの確率。実プレイで「忘れた頃に出る」程度の頻度を狙う
- ruins の HP を 3 にして、通常障害物（HP 2）より少し時間がかかるようにした。落下も遅めで撃つ時間を稼げる

### 採らなかった案

- **バリアをコイン購入式にする** — 操作が増える。落下取得が直感的
- **バリア中の被弾ごとに残時間を減らす消耗式** — 仕様を複雑にしない。時間で切るだけ
- **クールタイム中に取得したアイテムをストック** — バランス崩壊リスク
- **遺跡を破壊不能に** — それなら撃つ理由が無くなる

## 影響範囲

- `app/tools/penguin-shooter/ToolClient.tsx`
  - `Obstacle.kind` を追加し、ruins / normal を分岐
  - `BarrierItem` 型、state、refs、sync を追加
  - obstacle spawn に ruins 抽選
  - ruins 撃破時のバリアアイテム drop
  - barrier item の移動・取得ループ
  - `barrierActiveRef` / `barrierCooldownRef` のフレーム管理
  - `playerHit` 処理にバリアガードを追加
  - ObstacleView に ruins ビジュアル、player の周りに barrier リング
  - HUD にバリア残秒表示
- `docs/specs/tools/penguin-shooter.md` — 障害物セクションに古代遺跡、バリアアイテムセクションを追加

互換性:
- 既存の障害物挙動は不変（normal の destructible / solid はそのまま）
- 新規追加のみ。既存テスト/UAT への影響は最小

## 残課題

- 実プレイで出現率 0.07 と HP 3 のバランス確認（特に終盤ステージで遺跡が出ないと厳しいかもしれない）
- スマホ・キーボードどちらの操作でも取得が容易か確認
- バリアアイテム取得を逃すとアイテムは画面下端で消える。取得猶予として横幅をもう少し広げる案は要プレイテスト
- バリア中のショット強化やボム回復との重複は今は単独で動くだけだが、後で「2 段階強化」案を入れる余地は残す
