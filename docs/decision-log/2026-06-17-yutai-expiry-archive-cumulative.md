# 2026-06-17 優待期限台帳 累計はアーカイブ済みも含める

## 背景

- 対象 tool: 株主優待リスト `/tools/yutai-expiry`
- 前提: [2026-06-14 アーカイブ機能](./2026-06-14-yutai-expiry-archive.md) で、アーカイブ済みは**すべての集計から除外**していた。
- ユーザー要望: アーカイブしても「累計の金額」には加えたままにしたい。

## 今回決めたこと（2026-06-14 の集計ルールを一部改訂）

集計を「累計（生涯記録）」と「現在値」の2軸で扱い分ける。

### 累計（生涯記録）→ アーカイブ済みも含める
- 「もらった」（`grantedTotalYen`）
- 「使った」（`usedTotalYen`）
- 「失効」（`expiredTotalYen`）
- 一度確定した事実は退避しても記録として残す。これにより
  `もらった − 使った − 失効 = 未使用` の関係が累計側で閉じる。

### 現在値 → アーカイブ済みを除外（2026-06-14 から据え置き）
- 「未使用合計額」「今月失効」
- 各タブ件数（今月の未使用 / 期限切れ / 期限未設定 など）
- 「退避＝当面の判断材料から外す」という意図を維持する。

### 注記
- 累計3タイルに `title` ツールチップ「累計（アーカイブ済みも含む）」を付ける。
  画面がビジーにならないよう、タイル上に常時テキストは出さずツールチップに留める。

## 判断理由

- 「もらった/使った/失効」は一度確定すると変わらない生涯記録であり、退避（アーカイブ）は
  消費でも取り消しでもないため、累計から外すのは不自然だった。
- 累計3つを同じ扱い（アーカイブ込み）に揃えることで `もらった − 使った − 失効 = 未使用`
  の関係が保たれ、数字の整合が崩れない。

## 既知のズレ

- 「未使用のまま退避（remaining が残るのにアーカイブ）」した場合、累計はアーカイブ込み・
  「未使用合計額」はアーカイブ除外のため、`もらった − 使った − 失効` と画面の「未使用合計額」が
  その分（アーカイブ済みの未使用価値）だけズレる。失効・使い切り後にアーカイブする通常運用では一致する。

## 影響範囲

- `app/tools/yutai-expiry/ToolClient.tsx`
  - `grantedTotalYen` / `usedTotalYen` / `expiredTotalYen` から `isArchived` 除外を撤去
  - `StatTile` に optional `hint`（`title` ツールチップ）を追加し、累計3タイルへ付与
