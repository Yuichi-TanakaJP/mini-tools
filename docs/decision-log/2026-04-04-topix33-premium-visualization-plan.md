# 2026-04-04 TOPIX33 premium 可視化の見せ方方針

## 背景

- `TOPIX33業種` は現状、日次のサマリー、上昇 / 下落ランキング、全33業種一覧までを MVP として提供している
- 有料機能プレースホルダーは追加済みだが、どの market tool を premium 化すると価値が出やすいかは未整理だった
- 会話の中で、TOPIX33業種データを使った月次グラフや業種比較の見せ方を先に具体化しておきたいという要望が出た
- 関連画面:
  - `app/tools/topix33/`
  - `components/ShareButtons.tsx`

## 今回決めたこと

- TOPIX33 の premium 候補は、単日ランキングの延長ではなく「月間の流れを読める可視化」を主軸にする
- 初期の premium UI は、次の 3 ブロック構成を第一候補とする
  1. 今月の業種モメンタム要約
  2. 月間ヒートマップ
  3. 業種選択式の折れ線チャート
- 折れ線チャートは 33 業種の全表示を初期状態にせず、初期表示は「今月の上位 3 業種 + 下位 3 業種」を基本とする
- 月間比較の補助指標として、`累積騰落率`、`プラス日数`、`直近5営業日の強さ` を優先候補にする
- premium の価値訴求は「グラフが増えること」ではなく、「業種の強さの継続・反転・偏りを短時間で読めること」に置く

## 判断理由

### 1. 最初にヒートマップを置く理由

- 33 業種を月次で俯瞰するとき、色の面で見るほうが全体傾向をつかみやすい
- どの営業日に地合いが広がったか、どこで特定業種だけが強かったかを一目で把握しやすい
- 現在のランキング UI と役割が重なりにくく、premium として差が分かりやすい

### 2. 折れ線グラフは選択式にする理由

- 33 本を同時に重ねると凡例と線が過密になり、スマホではほぼ読めない
- 比較対象を絞れるようにしたほうが、「半導体寄り」「内需寄り」など自分の見たい軸に合わせやすい
- 初期表示を上位 / 下位の代表業種に限定すれば、初見でも画面の意味が伝わりやすい

### 3. premium は分析補助まで含める理由

- ただグラフを置くだけでは無料部分との差が弱い
- `今月いちばん安定して強い業種`、`直近で反転した業種` のような要約があると、データ閲覧から判断補助へ一段上げやすい
- 将来的に AI コメントや市場メモ連携へ広げる場合も、この要約ブロックが起点になる

### 4. 初期スコープで採らない案

- 33 業種の全折れ線同時表示
- 個別銘柄まで drilldown する詳細画面
- 3D 風表現やアニメーション中心の演出重視 UI

理由:

- 読みやすさより情報量や演出が勝ってしまい、日々の相場確認ツールとしては使いにくくなるため

## 推奨 UI 構成

### 1. 今月の業種モメンタム

- カード形式で 3 から 4 指標を見せる
- 候補:
  - 今月首位の業種
  - 今月最弱の業種
  - プラス日数最多の業種
  - 直近5営業日で急浮上した業種
- 1 行コメントを添える
  - 例: `銀行業は月初から上位維持。押し目が浅い`

### 2. 月間ヒートマップ

- 横軸を営業日、縦軸を 33 業種にする
- セル色は `chg_pct` ベースで、プラスは緑系、マイナスは赤系、中立は薄灰に寄せる
- デフォルト並び順は次のいずれか
  - 当月累積騰落率順
  - 当日騰落率順
- ホバー / タップで当日の騰落率と順位を表示する

### 3. 業種選択式の折れ線チャート

- 初期表示は `今月上位3 + 下位3`
- チェックボックスまたはトークン UI で業種を追加 / 削除できるようにする
- 表示本数は 6 本前後を推奨上限とし、多すぎる場合は注意表示を出す
- 値は `月初を100とした相対指数` か `月初来騰落率` のどちらかで表示する
- 初期案では「差の開き」が直感的に分かる `月初を100とした相対指数` を優先候補とする
- preview 実装では、初期表示を代表 4 業種に絞りつつ、最大 6 業種まで切り替えられる UI を先に試す
- preview 実装では、その月の最初の営業日を `100` に固定し、月内の営業日だけで相対推移を描画する
- preview 実装では、月送りで対象月を切り替えられるようにし、`全部表示` では TOPIX33 の全33業種を重ねて確認できるようにする
- preview 実装では、縦軸は `100` 中心固定ではなく、表示中系列の最大値 / 最小値を `5刻み` で丸めたレンジを使う
- preview 実装では、折れ線にホバーした際に業種名が分かるようにして、比較時の識別負荷を下げる

### 4. 月間ランキング表

- 日次ランキングの代替ではなく、月間評価の補助として置く
- カラム候補:
  - 月間累積騰落率
  - プラス日数
  - 最大連騰 / 最大連落
  - 直近5営業日騰落率
- テーブルからチャート対象に追加できる導線を用意する

## コピー方針

- premium タイトル案:
  - `業種モメンタム`
  - `月間セクタートレンド`
  - `業種の流れを見る`
- 補助コピー案:
  - `今月の強い業種と失速業種を、日次の積み上がりで確認`
  - `単日ランキングでは見えにくい、継続と反転をまとめて把握`

## 実装の優先順位

### Phase 1

- 月間ヒートマップ
- 月間サマリーカード

### Phase 2

- 業種選択式の折れ線チャート
- 月間ランキングとの連動

### Phase 3

- 要約コメントの自動生成
- 保存した比較セットやお気に入り業種の導線

## 影響範囲

- premium 機能の企画整理
- `app/tools/topix33/` の将来 UI 設計
- `components/ShareButtons.tsx` の premium 導線で何を訴求するか
- market tools 系 docs の今後の優先度整理

## 残課題

- 月間表示に使う基準値を `月初来騰落率` と `月初=100指数` のどちらで統一するか
- ヒートマップと折れ線チャートを同一ページに置くか、premium モーダル / 別ページに分けるか
- `TOPIX33` 以外の market tools と premium 価値をどう棲み分けるか
- 実装時に必要な月次集計ロジックを `mini-tools` 側で持つか、`market_info` 側で前計算するか
- 折れ線 hover を日付・値つき tooltip まで拡張するか
- `全部表示` 時の可読性を上げるため、線幅・凡例・ハイライト方法を再設計するか
- 業種選択 UI を、単純なトグルから「固定ピン」「最近見た業種保持」まで広げるか
- 月ごとに初期表示する業種を自動選定する現仕様を維持するか、ユーザー固定設定を優先するか
- ヒートマップを代表業種 preview から 33 業種フル表示へ広げるときの並び順とスクロール挙動をどうするか
- 今回の preview 実装は見せ方検証が目的のため、デザイン・操作性・ tooltip の磨き込みは別 Issue で継続する
