# My Stocks Ratio Chart Types

Date: 2026-06-14

## Decision

`my-stocks` の `保有メモ` → `比率` 表示に、グラフ種別の切り替えトグル（`帯グラフ` / `円グラフ` / `四角ヒートマップ`）を追加する。

- `帯グラフ`: 従来の積み上げ帯（`StackedShareBar`）。
- `円グラフ`: SVG のドーナツ（`DonutChart`）。`stroke-dasharray` で各区分を 1 周 100 に正規化して描画する。
- `四角ヒートマップ`: squarified treemap（`TreemapChart` + `squarifyTreemap`）。各四角形のアスペクト比が 1 に近づくよう領域分割し、面積で比率を表す。

口座別・銘柄別のどちらのパネルも同じトグルで表示が切り替わる。各グラフの下には正確な %/金額を示す凡例（`ShareLegend` → `ShareRow`）を常に表示する。

加えて「比率の対象」トグル（`取得額` / `配当（年間・推定）`）を設け、同じグラフ群を取得額ベース・配当ベースで切り替えられる（`SegmentedTabs` に共通化）。配当ベースの値は `数量 × dividendByCode[code].perShare`（推定年間配当 1 株あたり）で算出する。

銘柄別は配色数（`STOCK_CHART_COLORS`、16 色）まで個別表示し、それを超えた末端のみ集約する。集約対象が 1 銘柄だけのときは集約せずその銘柄をそのまま表示し、2 銘柄以上のときだけ `他 N 銘柄` にまとめる。いずれの場合も合計は全体と一致する。

## Reason

取得額比率は口座区分・銘柄数が増えると帯グラフだけでは内訳を直感的に掴みにくい。円グラフは構成比、ヒートマップは銘柄数が多いときの面積比較に向くため、同じデータを複数の見せ方で切り替えられるようにした。

ライブラリは追加せず、SVG と CSS（絶対配置 + `aspect-ratio`）だけで描画する。端末内データのみを扱う既存方針・依存の軽さを維持するため。

配当は既存の `MyStocksReference.dividendByCode` をそのまま使うため新規 API 取得は不要。取得額と同じグラフ群を再利用でき、ポートフォリオの「金額の偏り」と「配当収入の偏り」を同じ操作感で見比べられる。`他 N 銘柄` への集約は配色数を上限にして、まとめすぎて個別銘柄が見えなくなる問題を避ける。

## Impact

- `比率` タブは `取得額` / `配当（年間・推定）` を切り替えて、口座別・銘柄別の構成比を 3 種類のグラフで表示できる。
- 配当ベースは「数量入力済み かつ 配当データあり」の銘柄が対象。除外件数と推定である旨の注記を表示する。
- 銘柄別は最大 16 銘柄を個別表示し、末端のみ（2 銘柄以上のとき）集約する。`他 N 銘柄` も含め合計は常に全体と一致する。
- グラフ種別・比率対象のトグルは `SegmentedTabs` に共通化し、`保有メモ` の表示モード切替（`SegmentedMode`）と別物として並存する。
