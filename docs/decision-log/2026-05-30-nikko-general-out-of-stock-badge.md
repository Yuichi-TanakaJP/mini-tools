# 2026-05-30 日興信用バッジの短語化と「売可だが在庫0」追加

## 背景

- `yutai-candidates` の日興信用バッジは `canNikkoGeneralCrossNow`（`general_short && available_shares>0 && 売建停止でない`）を一般売りの基準にしている。
- このため `general_short=true` でも `available_shares=0` の銘柄はどのバッジにも該当せず、画面上「一般売りの情報が一切ない」状態に見えていた。
- 「数量0は売り不可ではなく、売建自体は可能で在庫残数が0なだけ」という指摘を受け、表示方針を再確認した。
- あわせて、`一般売可 / 一般注意 / 一般規制 / 制度売可` のラベルが冗長で見栄えが悪いため、ラベル体系全体を見直した。

## 今回決めたこと

- 日興信用バッジを短語化する。
  - `一般売可` → `一般可`
  - `一般注意`（貸株注意喚起） → `一般規制`
  - 売建規制・取引停止 → `一般×`（`一般規制` より重い状態）
  - `制度売可` → `制度可`
- 新たに `general_short=true` かつ `available_shares===0` かつ売建停止でない状態を `一般—`（売可だが在庫0、今クロス不可）として表示する。
- 判定は `isNikkoGeneralOutOfStock` に切り出し、表示順は `一般× / 一般規制 / 一般可 / 一般—` の排他チェーン、`制度可` は別枠。
- 各バッジの詳細（中身が貸株注意喚起か取引停止か等）は `title` 属性で補足する。
- `available_shares===null`（在庫不明）は対象外とし、従来どおり画面に推測ラベルを出さない。

## 判断理由

- `available_shares>0` を一般売りの必須条件にしていたのは「今クロス約定できるか」を示すためで、その判断自体は維持する。
- 一方で「売建可否」と「在庫残数」をまとめて非表示にすると、売可だが在庫切れの銘柄が制度規制銘柄と区別できず、誤解を招いていた。`一般—` を別バッジにすることで「今は約定できないが売建自体は可能」を可視化する。
- `一般`/`制度` をプレフィックスに残しつつ後半を `可 / 規制 / × / —` の短語に揃えることで、一覧で読みやすくする。記号 `○△` は見栄えが悪いため使わない。意味の詳細は `title` で補う。
- `一般規制`（中身は貸株注意喚起）と `一般×`（取引停止）が並ぶと severity の語感が逆に見えるが、`×` を最重とする運用とし、`title` と docs で補足する。
- `一般—` には muted 系の既存スタイル `creditChipNoCross` を流用し、緑（`一般可`）と視覚的に区別する。
- `null`（在庫不明）まで `一般—` に含めると「在庫0」と「在庫不明」を混同するため、`=== 0` で厳密に絞る。

## 影響範囲

- `app/tools/yutai-candidates/ToolClient.tsx`
  - `isNikkoGeneralOutOfStock` 追加
  - `renderCreditBadges` を短語ラベル＋`title` に変更し、`一般—` を排他チェーンに追加
  - 未使用だった `creditChipNoCross` スタイルを使用
- `docs/specs/cross-cutting/market-tools-data-fetch-paths.md` の badge 判定表・凡例を更新
- 既存の `canNikkoGeneralCrossNow` とフィルタ（`日興: 一般売可` 等の select 文言）の挙動は不変。`一般—` 銘柄は「今クロス可」ではないため `general` フィルタには含めない。

## 残課題

- ラベルの詳細は `title`（hover）でしか出ないため、Android などタップ環境向けの凡例表示が要るかは今後判断する。`一般規制` と `一般×` の severity 関係も凡例で明示するか検討する。
- `一般—` をフィルタ条件（在庫復活待ちの監視リスト）として独立させるかは今回決めない。
