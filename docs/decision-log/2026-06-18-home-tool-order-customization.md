# 2026-06-18 ホーム画面のツール配置カスタマイズ

## 背景

- ホーム画面のツール数が増え、ユーザーごとによく使うツールが異なる状態になっている。
- 固定順のままだと、よく使うツールへ到達するためのスクロール量が増える。

## 今回決めたこと

- ホーム画面のツール一覧に「配置変更」モードを追加する。
- スマホではカードのロングタップでも「配置変更」モードへ入れるようにする。
- 配置変更中はカードをリンクとして開かず、上下ボタンで順番を変更できるようにする。
- 初期順は「マイ銘柄リスト」を先頭にし、「合計計算」「文字数カウント」は末尾へ移動する。
- 変更後の順番は `mini_tools_home_tool_order_v1` に `href` 配列として LocalStorage 保存する。
- 初期順へ戻す操作を用意し、保存値を削除できるようにする。

## 判断理由

- ユーザーごとの好みはサーバー DB ではなく端末内保存で十分で、既存の localStorage 方針とも合う。
- `href` 配列だけを保存すれば、ツールの文言や説明変更に追随しやすく、新しいツールは保存順の末尾へ自然に追加できる。
- ドラッグ専用 UI ではなく上下ボタンにすることで、モバイルやキーボード操作でも同じ操作を提供できる。
- スマホのロングタップは「編集開始」のショートカットに限定し、移動自体は上下ボタンで行う。スクロール中の誤発火を避けるため、押下中に一定以上動いた場合はロングタップ判定をキャンセルする。

## 影響範囲

- 対象画面: ホーム画面 `/`
- 対象ファイル: `app/ToolGridClient.tsx`, `lib/tools-catalog.ts`, `docs/product-spec.md`
- 保存先: ブラウザ LocalStorage の `mini_tools_home_tool_order_v1`
- 既存ユーザーの保存済み配置は維持する。初期順変更は LocalStorage に保存順がないユーザー、または初期順へ戻したユーザーに反映される。

## 残課題

- ドラッグアンドドロップでカードを直接つかんで移動する挙動は今回スコープ外。
- 複数端末間の同期は行わない。
