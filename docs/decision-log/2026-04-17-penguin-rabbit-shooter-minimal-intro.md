# 2026-04-17 ペンギン・バニーシューター最短導入方針

## 背景

- `mini-tools` におまけ要素として小さなゲームを追加したい相談があり、まずは最短ルートで試すことになった
- 共有された試作コードは `shadcn/ui` 前提だったが、この repo は同構成を採っていない

## 今回決めたこと

- 初回導入は `app/tools/penguin-rabbit-shooter/` に独立した tool として追加する
- キャラクターや演出は外部画像ではなく絵文字を使い、追加アセットなしで成立させる
- UI は既存の `mini-tools` スタイルに合わせ、`shadcn/ui` は新規導入しない
- スマホではキーボード前提にせず、画面下のタッチボタンで移動と発射を行えるようにする
- 縦長スマホではゲーム面を高さ制限つきで縮小し、操作パネル込みで 1 画面に収まりやすいレイアウトを優先する
- 初期描画で SSR と client の差分が出ないよう、装飾用の星配置も決定的に生成する

## 判断理由

- 最短で公開可能な形を作り、ゲーム導入の相性や導線の見え方を先に確認したかった
- 画像アセット管理や UI ライブラリ導入を同時に行うと、判断軸が増えて初回追加のコストが上がる
- 既存 repo の tool 追加パターンに寄せた方が保守しやすい
- スマホ閲覧だけできて操作できない状態だと、おまけ tool としての体験が成立しにくい
- 縦長スマホではゲーム面を横幅いっぱいに保つより、操作パネルを含めた全体の視認性を優先した方が試しやすい
- `Math.random()` 依存の初期描画は hydration mismatch の原因になるため、SSR 前提の初期状態では避ける

## 影響範囲

- `app/tools/penguin-rabbit-shooter/` の新規追加
- トップページの tool 一覧への導線追加
- 今後アセット差し替えを行う場合は、`public/` 配下の画像管理方針を別途決める余地がある

## 残課題

- トップページ上で「投資系 tool」と「おまけ tool」を視覚的に分けるかは未決定
- 効果音や画像素材の導入は未対応
