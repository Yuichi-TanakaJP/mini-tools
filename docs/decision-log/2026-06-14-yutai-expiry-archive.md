# 2026-06-14 優待期限台帳 アーカイブ機能

> **改訂あり**: 集計ルールは [2026-06-17 累計はアーカイブ済みも含める](./2026-06-17-yutai-expiry-archive-cumulative.md) で一部更新。累計（もらった/使った/失効）はアーカイブ済みも含める。

## 背景

- 対象 tool: 株主優待リスト `/tools/yutai-expiry`
- 期限切れ・使用済みの優待が一覧に残り続け、最新の使える優待が埋もれるという要望。
- ユーザー要望: 「アーカイブ選択」と「アーカイブを含むかどうかの選択」を両方できるようにしたい。
- 関連: [2026-05-20 消費モデル（枚数/金額・履歴）](./2026-05-20-yutai-expiry-consumption-model.md)

## 今回決めたこと

### データモデル（`BenefitItemV2` に optional 追加・後方互換）
- `archivedAt: string | null` を追加（ISO 文字列＝アーカイブ日時 / null＝未アーカイブ）。
- bool ではなく日時にしたのは、後から「いつ退避したか」を表示・並べ替えに使える余地を残すため。判定は `!!archivedAt`。
- `coerceItem` で正規化（不正値は null）。旧データ・エクスポート JSON は `archivedAt` 無しでも安全に読める（破壊的変更なし）。

### 操作（純関数）
- `setArchived(item, archived)`：`archivedAt` のみ更新。残量・履歴・`isUsed` は変えない（アーカイブは「退避」であり消費ではない）。

### UI / 表示ルール
- カード / リストの各行に「アーカイブ」「アーカイブ解除」操作を追加。
- 「アーカイブを含む」トグルを `使用済含む` の隣（desktop / mobile 両方）に追加。**既定 OFF**。
- 既定ではアーカイブ済みを **一覧・タブ件数・金額集計（もらった/使った/失効/未使用合計など）から除外**する。`使用済` とは独立した軸として扱う。
- 「アーカイブを含む」ON のときだけ一覧に表示。アーカイブ済みは「アーカイブ済」バッジ＋淡色（`cardUsed`/`rowUsed`）で区別。
- アーカイブ / 解除の直後は Undo トーストを出し、誤操作を可逆にする（履歴集計の net / Undo Toast 方針に揃える）。

## 判断理由

- `isUsed`（消費の有無）と `archivedAt`（一覧から退避するか）は意味が異なるため、別フィールドにして直交させた。これにより「未使用だが期限切れで退避したい」も表現できる。
- 既定で集計からも除外するのは、「退避＝最新の判断材料から外す」というユーザー意図に合わせ、トグルの ON/OFF で見える数字が直感的に一致するようにするため。
- 削除ではなくアーカイブにすることで、過去にもらった優待の記録を残しつつ一覧をすっきりさせられる。

## 影響範囲

- `app/tools/yutai-expiry/benefits/store.ts`（型・`coerceItem` 正規化・`setArchived`）
- `app/tools/yutai-expiry/ToolClient.tsx`（`showArchived` トグル、filter、件数/集計の除外、カード/テーブルの操作・バッジ）
- 互換性: 旧 v1 / 旧 v2 / 旧エクスポート JSON はそのまま取り込み（`archivedAt` 既定 null）。

## 残課題

- アーカイブ専用タブやアーカイブ日時での並び替えは今回スコープ外。必要になれば `archivedAt` を活用して追加する。
