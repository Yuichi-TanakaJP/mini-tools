# 2026-06-26 ホーム通知に今日・明日の市場イベントを追加

## 背景

- ホーム通知の初期実装として、開示イベントレーダーの未確認通知を追加した。
- 次の通知候補として、毎朝の確認価値が高い「今日・明日の注目イベント」をホームに出す案が出た。
- 決算カレンダーは国内・海外の決算予定を既に日付単位で持っている。

## 今回決めたこと

- ホーム通知に、今日・明日の決算予定を追加する。
- 表示は国内件数・海外件数の日別サマリーと、端末内マイ銘柄に一致する決算予定の一部にする。
- マイ銘柄照合はブラウザ内で行い、登録銘柄コードは API へ送信しない。
- ホーム通知用 API `/api/earnings-calendar/notifications` は、今日・明日の国内/海外予定を広めに返す。
- ホーム通知用 API `/api/econ-calendar/notifications` は、今日・明日の重要経済指標だけを返す。
- 決算通知・経済指標通知には既読状態を持たせず、当面は「今日・明日の予定」として表示する。
- 開示通知と決算通知は独立に取得し、片方の取得失敗でホーム通知全体を落とさない。

## 判断理由

- 今日・明日の決算予定は、ホーム画面を開く動機になりやすい。
- 既存の決算カレンダー data-loader を API route から再利用でき、表示ロジックを重くしすぎずに始められる。
- マイ銘柄の照合を端末内に残すことで、既存のローカル保存方針を維持できる。
- 決算予定と経済指標は日付で自然に消えるため、最初から既読管理を入れなくても実用上のノイズを抑えやすい。
- 経済指標は件数が多くなりやすいため、最初は `impact >= 4` に絞る。

## 影響範囲

- ホーム画面 `/`
- `app/HomeNotifications.tsx`

- `app/api/econ-calendar/notifications/route.ts`
- `docs/product-spec.md`
- `docs/specs/tools/earnings-calendar.md`

## 残課題

- 決算カレンダー・経済指標カレンダー側に日付指定 query を追加し、ホーム通知から今日・明日へ直接ジャンプできるようにする。
- 決算集中日の閾値や「注目」の定義を設定可能にする。
- 決算通知・経済指標通知を非表示にするユーザー設定の要否を検討する。
