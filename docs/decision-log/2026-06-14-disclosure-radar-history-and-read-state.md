# 2026-06-14 開示レーダーの履歴表示と確認済み管理

## 背景

- 優待変更は毎日発生するとは限らず、最新1日だけでは見落としやすい。
- LINE通知から開いたときに対象イベントをすぐ確認できる導線が必要。

## 決定

- 初期表示を直近7暦日とし、1日・7日・30日を切り替え可能にする。
- 1日表示はmanifestの最大日ではなく、`manifest.latest` の非0件日を使う。
- 最大30暦日分をServer Componentで取得し、期間・分類・検索はブラウザ内で絞り込む。
- 確認済みイベントIDはlocalStorageへ保存し、公開元リンクを開いた場合も確認済みにする。
- LINEはイベントID付きURLを生成し、対象カードを強調してスクロールする。

## 理由

- 7日表示なら週末や開示のない日をまたいでも実用的な一覧になる。
- 個人の確認状態をサーバーへ送らず、既存の端末内保存方針を維持できる。
- APIの既存manifest・日付指定契約だけで実現でき、API変更を増やさずに済む。
