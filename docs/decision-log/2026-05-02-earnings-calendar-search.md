# 2026-05-02 決算カレンダーへの銘柄検索機能追加

## 背景

月間カレンダーを日付ベースで見ているとき、特定銘柄の決算日を調べたい場合に月を一つずつめくる必要があった。
銘柄名やコードで横断検索できれば、目的の銘柄にすばやく到達できる。

## 今回決めたこと

- ヒーローセクション直下に常時表示される検索バーを追加する
- 検索クエリがある間はカレンダーを隠して結果リストを表示する（検索 / 通常を排他的に切り替え）
- 検索は国内・海外の全月データを横断する（アクティブなタブに関わらず両方対象）
- 結果タップで対象日に遷移して検索をクリアし、通常カレンダーに戻る
- 最大60件でクリップする（パフォーマンス上限）

## 判断理由

**検索中はカレンダーを隠す（排他表示）**
- 検索結果と月間カレンダーを同時に見せると画面が長くなりすぎる
- モバイル幅（max 440px）では特に縦スクロールが増えて使いにくい
- 検索の目的は「目的の銘柄の日付を知ること」なので、カレンダーは遷移後に見れば十分

**全市場横断検索**
- ユーザーは銘柄を特定してから市場タブを絞り込む流れではなく、銘柄名から検索する流れが自然
- タブを切り替えてから検索するUXより「どの市場でも入力すれば出てくる」方が直感的

**60件制限**
- インクリメンタルサーチのたびに全データを走査するため、上限がないと広い範囲で重複ヒットしたとき遅延する可能性がある
- 60件あれば実用上ほぼ足りる（検索を絞ればすぐ数件に収まる）

## 影響範囲

- `app/tools/earnings-calendar/ToolClient.tsx` のみ変更
- データ取得・型・APIには変更なし
- 新規追加 state: `searchQuery`, `searchResults`
- 新規追加 UI スタイル: `searchBar`, `searchInputWrap`, `searchIcon`, `searchInput`, `searchClear`, `searchResultsHeader`, `searchResultsCount`, `searchResultDate`

## 残課題

- 検索結果表示時にページトップへのスクロールを行っていない（長いリストを見た後に検索するとUXが悪い可能性）
- 検索 → カレンダー遷移時のアニメーションなし（フラッシュが発生する場合あり）
- スクロール復元が未実装
