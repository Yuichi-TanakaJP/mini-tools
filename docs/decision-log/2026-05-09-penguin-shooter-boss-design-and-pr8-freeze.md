# 2026-05-09 ペンギンシューター ボス設計とPR8凍結

## 背景

- ペンギンシューターは5大ステージ x 20小ステージのデータ構造まで追加済み
- 次の開発対象は PR 7「ボス設計」
- その次に予定していた PR 8「2人プレイと進捗保存」は、ユーザー判断でいったん凍結する

## 決めたこと

- PR 7では、各大ステージの中間ボス / ステージボスを個別データ化する
- ボスごとに以下を持つ
  - 名前
  - HP
  - スコア
  - 報酬コイン
  - 攻撃ラベル
  - 速度補正
  - 横移動補正
- ボス名と攻撃ラベルを画面上に表示する
- 最終ステージボス撃破は Shoot 救出と世界平和へつながる最終クリアとして扱う
- PR 8「2人プレイと進捗保存」は凍結し、実装順から外す

## 判断理由

- ボス設計は100小ステージ構成のゲーム性を強くするため、先に進める価値が高い
- 2人プレイと保存は操作体系、保存データ、既存プレイ体験への影響が大きいため、ボス体験が固まるまで凍結する
- 凍結をdocsに残すことで、次の作業が「PR 8へ自動的に進む」状態にならないようにする

## 影響範囲

- `app/tools/penguin-shooter/stageData.ts`
- `app/tools/penguin-shooter/ToolClient.tsx`
- `docs/plans/penguin-shooter-new-game-plan.md`

## 残課題

- ボス固有の攻撃弾や弾幕を追加する
- ボスごとの専用演出や撃破アニメーションを強化する
- 世界平和エンディングを専用演出にする
- PR 8を再開する条件を別途決める
