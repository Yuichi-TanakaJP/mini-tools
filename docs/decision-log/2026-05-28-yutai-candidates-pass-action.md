# 2026-05-28 yutai-candidates の「パス」アクションとカードレイアウト見直し

## 背景

- 優待カレンダー（`/tools/yutai-candidates`）には「ピック」（気になる銘柄の一時マーク）はあったが、「これは興味ない／今期は見送り」を残す手段がなかった。
- 結果としてスクロールするたびに同じ銘柄を再評価することになり、検討対象の絞り込みが進まない。
- 既存カードのフッターは `みんかぶ｜企業サイト｜ピック｜メモ追加` の 4 枠で、これに 5 枠目を足すと破綻する。

## 今回決めたこと

- 「逆ピック」相当のアクションを **「パス」** という名称で導入する。
  - ピック（取る）の対義語として自然で、トランプ的に「今回はパス」のニュアンスが伝わる。
  - 「除外」「ミュート」「スキップ」も検討したが、永続非表示の語感が強いものや事務的なものは避けた。
- パスとピックは **排他** にする（パスを付けたらピックは外す／逆も同じ）。
  - 「気になる」と「興味ない」が同居する状態は意味がないため。
  - `added`（優待メモ追加済み）とは独立に扱う。
- カード右上の `stateChips` 位置にピックを **★ アイコンボタン** として移し、フッターには「パス」を入れる。
  - フッター構成: `みんかぶ｜企業サイト｜パス｜メモ追加`
  - お気に入りパターン（★ = 気になる）に寄せ、目立つアクションを上に、消極アクションをフッターに配置する。
- パス済みカードは灰色＋透過（opacity 0.62）で「処理済み感」を出し、薄く残す。
  - 完全に非表示ではなく薄く残すことで、誤操作からの復帰や見直しを許容する。
  - 隠したい場合は状態フィルタ「未選択」を使えば除外できる。
- 状態フィルタに `passed` を追加し、`unselected` は picked / passed / added のいずれにも該当しないものとする。
- 保存先 localStorage キー: `monthly_yutai_passes_v1`（picks と分離）。

## 判断理由

- 名称「パス」: ピックの対義として最も短く、再開可能（解除できる）ニュアンスが残る。
- ★ アイコン移動: フッター枠を増やさずアクションを 1 つ追加でき、視覚的にも「気になる＝目立つ位置」が自然。
- 排他: ピック／パスは「同じ銘柄に対する評価」なので、共存させると状態管理も UI もぼやける。
- 薄く残す表示: 「興味ない」と決めた銘柄を完全消去すると誤操作のリカバリが面倒。グレーアウトはお気に入り解除と同様、可逆な選択肢を視覚的に保つ。
