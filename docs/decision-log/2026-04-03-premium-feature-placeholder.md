# 2026-04-03 有料機能プレースホルダーの追加方針

## 背景

- 将来の有料機能拡張を見据え、UI 上に「準備中」を示すプレースホルダーを置きたいという要望が出た
- 現状は全機能が無料・ログイン不要で動作しているが、課金機能を後から追加できる設計にしておく
- 関連コンポーネント: `components/ShareButtons.tsx`

## 今回決めたこと

- `ShareButtons` に `"premium"` という新しい `ShareMethod` を追加した
- QR ボタンの左横に星アイコン（金色）のボタンを表示する
- ボタンは `disabled` 状態で表示し、ホバー時に「プレミアム機能（準備中）」とツールチップを出す
- `DEFAULT_METHODS` に `"premium"` を含めることで全ツールに自動表示される

## 判断理由

- **なぜ QR の左横か**: 既存ボタン列の末尾（コピー → QR）の間に挟むことで、QR が右端に残り並び順が自然
- **なぜ disabled にするか**: クリック可能に見せると UX を損なう。disabled + tooltip で「まだ使えない」を明示する
- **なぜ星アイコンか**: 王冠より汎用的で SVG が軽量。プレミアム感を十分に伝えられる
- **なぜ金色（#f59e0b）か**: 他のボタン（モノクロ）と差別化し、将来的に目立つ機能と認識させるため

## 影響範囲

- `components/ShareButtons.tsx`: `ShareMethod` 型・`DEFAULT_METHODS`・`renderIcon`・`labelFor`・`methods.map()` を変更
- 全ツールの共有ボタン UI に premium ボタンが追加される（デフォルト表示）
- 既存のシェア機能には変更なし

## 残課題

- ログイン機能の実装が必要（Auth.js / NextAuth v5 が候補）
- 課金システムの検討（Stripe が第一候補）
- 有料にする具体的な機能を決める（現時点では未定）
- ログイン後の premium ボタンの動作設計（機能解放 or 別ページ遷移など）
- 未ログイン時のクリックで「ログインが必要」モーダルを出す実装
