# 2026-05-09 ペンギンシューター 100小ステージデータ構造

## 背景

- ペンギンシューターは5大ステージ x 20小ステージ、合計100小ステージの豪華版ゲームとして育てる方針になった
- 直前の実装では100小ステージ相当の進行値は持っていたが、ステージ名、背景、ボス地点、難易度、報酬が `ToolClient.tsx` 内に散らばりやすい状態だった

## 決めたこと

- 100小ステージの基本データは `app/tools/penguin-shooter/stageData.ts` に分離する
- 各大ステージは以下のデータを持つ
  - ステージ番号
  - ID
  - 表示名
  - ストーリー表示名
  - 背景
  - アクセント色
  - BGM生成音の基準周波数
  - 中間ボス名
  - ステージボス名
  - 20小ステージの設定
- 各小ステージは以下のデータを持つ
  - 大ステージ内番号
  - 全体通し番号
  - 表示ラベル
  - 敵出現間隔
  - 敵速度補正
  - 報酬コイン数
  - ボス地点
- 10小ステージ目を中間ボス、20小ステージ目をステージボスとして扱う
- 10小ステージクリアで2人プレイ解放状態をUI表示する

## 判断理由

- 今後のボス個性化、敵構成、報酬、保存データを足す前に、ステージ進行の土台を構造化しておく必要がある
- `ToolClient.tsx` は描画とゲームループに集中させ、ゲーム内容の設定はデータ側へ寄せる方が拡張しやすい
- 2人プレイは操作実装より先に解放状態を見える化し、次のPRで操作・保存へ進めやすくする

## 音データの扱い

- 現在のBGM/効果音は音声ファイルではなく、Web Audio API で生成している
- そのため `public/games/penguin-shooter/` 配下には現時点で背景SVGのみがある
- 将来ファイル音源へ置き換える場合は、`public/games/penguin-shooter/audio/` 配下を候補にする

## 影響範囲

- `app/tools/penguin-shooter/ToolClient.tsx`
- `app/tools/penguin-shooter/stageData.ts`
- `docs/plans/penguin-shooter-new-game-plan.md`
- `docs/specs/tools/penguin-shooter.md`
- `docs/uat/penguin-shooter.md`

## 残課題

- 各大ステージごとの敵編成をさらに細かく分ける
- 中間ボス/ステージボスの攻撃、HP、専用演出を追加する
- 2人プレイの操作方法と保存仕様を決める
- ファイル音源を採用する場合の形式、音量、ライセンス、差し替え手順を決める
