# mini-tools プロダクト仕様

このドキュメントは、`mini-tools` の現在仕様を俯瞰するための入口です。
詳細な画面仕様は [ツール別仕様](./specs/index.md)、動作確認観点は [UAT チェックリスト](./uat/index.md)、仕様判断の理由は [Decision Log](./decision-log/) を参照します。

## 目的

`mini-tools` は、日々の投資・株主優待・市場確認・小さな計算や入力作業を、軽量な Web ツールとしてまとめる単一アプリです。

主な目的:

- よく使う確認作業をブラウザで素早く実行できるようにする
- 入力系ツールは端末ローカルで完結させ、個人メモをサーバーに送らない
- 市場データ系ツールは同梱 JSON と外部 API を使い分け、データ更新と表示を分離する
- アジャイルに拡張しながら、現在仕様と判断履歴を docs で追えるようにする

## 対象ユーザー

- 日本株・米国株・株主優待の情報を日常的に確認する個人ユーザー
- 端末内で小さなメモや入力補助を完結させたいユーザー
- 開発・運用上は、mini-tools の仕様、確認観点、データ取得経路を後から追いたい開発者

## ツール分類

| 分類 | 主なツール | 保存・取得方針 |
|---|---|---|
| LocalStorage 系 | 優待銘柄メモ帳、株主優待期限帳、文字数カウント、合計ツール | ブラウザ内保存、または保存なし |
| 国内市場データ系 | 株価ランキング、日経225寄与度、TOPIX33業種、優待カレンダー、決算カレンダー国内 | repo 同梱 JSON と market-info API をツールごとに使い分ける |
| 海外・API 系 | 米国株ランキング、市場ランキング、決算カレンダー海外、経済カレンダー、EDINET 書類 | 主に `MARKET_INFO_API_BASE_URL` 経由の API |
| premium preview 系 | premium ページ、premium 表示枠 | Cookie ベースの仮ログイン。会員 DB・課金は未実装 |
| ゲーム / 実験系 | ペンギン・エイリアンシューター | 単体のクライアント体験として扱う |

## 共通仕様

### ルーティング

- Next.js App Router を使う
- 各ツールは原則 `app/tools/<tool>/page.tsx` を入口にする
- ツール URL は `/tools/<tool>` 形式にする
- 共有・canonical URL は各ページの `metadata.alternates.canonical` で明示する

### UI

- 全ページ共通のヘッダーを使う
- ツール固有の UI は `ToolClient.tsx` などに分ける
- 入力系・操作系は Client Component を使う
- 初回データ取得やアクセス制御が必要な画面は Server Component で初期データを組み立てる
- ホーム画面のツール一覧は、ユーザーが並び順を変更できる。配置変更ボタンに加え、スマホではカードのロングタップでも配置変更モードへ入れる。変更後の順番はブラウザの LocalStorage に保存し、サーバーには送信しない
- ホーム画面の通知は、まず画面内の通知カードとして表示する。対象は株主優待期限帳の未使用・未アーカイブで7日以内に期限を迎えるアイテム、開示イベントレーダーの直近7日間の優待系未確認イベント、端末内マイ銘柄に一致する未確認イベント、今日・明日の国内/海外決算予定（保有/ウォッチ銘柄と日経225採用銘柄を強調）、今日・明日の重要経済指標とする。優待期限と保有/ウォッチ銘柄の照合はブラウザ内で行い、登録内容はサーバーへ送信しない

### データ取得

- 市場データ系は、ツールごとの `data-loader.ts` を中心にデータ取得をまとめる
- API の base URL は `MARKET_INFO_API_BASE_URL` を使う
- API 障害時の fallback 有無はツールごとに仕様へ明記する
- 詳細は [Market Tools データ取得経路一覧](./specs/cross-cutting/market-tools-data-fetch-paths.md) を参照する

### 保存

- 個人メモや入力途中データは、原則として LocalStorage に保存する
- LocalStorage 系ツールは SSR 時にブラウザ保存を読まない
- サーバー DB は現時点では導入していない

### エラー・空状態

- 外部 API 失敗時に raw error を画面へ露出しない
- データなし、未設定、取得失敗は、ツールごとに空状態またはメッセージで扱う
- LocalStorage が使えない場合もページ全体をクラッシュさせない

### premium

- premium は現時点では preview 導線
- Cookie ベースの簡易ログインを使う
- 本格的なユーザー管理、課金、権限テーブルは未実装

- 確認観点が変わったら、UAT チェックリストも更新する
